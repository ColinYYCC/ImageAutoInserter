## 项目结构

本项目包含两个独立但相关的部分：

### 1. 核心功能（CLI 版本 v1.x）✅ 已完成

**位置**: `.trae/specs/image-auto-inserter/`

这是项目的**核心功能**，负责实际的图片处理和 Excel 插入操作。

**核心能力**:
- 从文件夹/ZIP/RAR 读取图片
- 自动识别商品编码
- 动态识别 Excel 表头位置
- 智能添加 Picture 列
- 批量嵌入图片到 Excel 单元格
- 错误处理和日志记录

**相关文档**:
- [`spec.md`](spec.md) - 核心功能规格说明
- [`tasks.md`](tasks.md) - 任务清单（阶段 1-6）
- [`checklist.md`](checklist.md) - 验收清单

---

### 2. 用户界面（GUI 版本 v3.x）🔄 开发中

**位置**: `.trae/specs/gui-redesign/`

这是**用户操作界面**，让用户通过图形界面调用核心功能。

**核心能力**:
- 文件选择器（Excel 和图片包）
- 进度条显示
- 结果展示
- 错误对话框
- 高级视觉设计（色彩/间距/圆角/动画系统）

**相关文档**:
- [`spec.md`](gui-redesign/spec.md) - GUI 界面设计规格
- [`tasks.md`](gui-redesign/tasks.md) - GUI 开发任务
- [`checklist.md`](gui-redesign/checklist.md) - GUI 验收清单

## 文档组织说明

### 为什么分开管理？

1. **职责分离**: 核心功能 vs 用户界面
2. **独立演进**: CLI 可独立于 GUI 存在和使用
3. **清晰边界**: 避免功能逻辑和界面设计混淆
4. **便于维护**: 修改界面不影响核心功能文档

### 何时需要同时查看？

- 开发 GUI 时：需要理解如何调用核心功能
- 修改核心功能时：需要确保接口向后兼容
- 验收测试时：需要验证整体流程
