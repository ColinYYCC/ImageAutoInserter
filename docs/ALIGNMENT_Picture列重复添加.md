# ALIGNMENT_Picture列重复添加.md

## 1. 问题描述

### 问题现象
- Excel 文件（如 `PL-26SDR-0076V3_含图.xlsx`）已包含 `Picture 1` 列
- 程序处理时仍然添加了新的 `Picture 1/2/3` 列
- 导致列重复、数据错位

### 影响范围
- 已处理过并包含 Picture 列的 Excel 文件
- 重新运行时会产生重复列

---

## 2. 根因分析

### 直接原因
`add_picture_columns()` 方法没有检查列是否已存在

### 代码层面
- **已有工具方法** `_column_exists()` (excel_processor.py:255) 用于检查列是否存在
- **未被调用**: `add_picture_columns()` 方法直接添加列，未调用检查方法

### 逻辑流程（修改前）
```
1. 读取 Excel
2. 直接调用 add_picture_columns()
3. 在末尾添加 Picture 1/2/3 列 ← 没有检查是否已存在
4. 嵌入图片
5. 保存
```

---

## 3. 修复方案

### 核心修改
修改 `add_picture_columns()` 方法逻辑：

| 步骤 | 修改前 | 修改后 |
|------|--------|--------|
| 1 | 直接添加列 | 先检查 Picture 列是否已存在 |
| 2 | 无条件覆盖 | 已存在则返回现有列号 |
| 3 | - | 不存在才添加新列 |

### 修复后逻辑流程
```
1. 读取 Excel
2. 调用 add_picture_columns()
   2.1 遍历 Picture 1/2/3 检查是否已存在
   2.2 已存在 → 记录列号
   2.3 全部存在 → 直接返回
   2.4 部分不存在 → 在末尾添加缺失列
3. 嵌入图片
4. 保存
```
