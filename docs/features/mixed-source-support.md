# 混合圖片來源支持

## 功能概述

ImageAutoInserter v2.0 現已支持同時選擇文件夾和 ZIP 文件作為圖片來源，實現統一的資源管理。

## 支持的圖片來源類型

### 📁 文件夾

直接選擇包含圖片的文件夾，系統會自動掃描文件夾中的所有圖片文件。

**支持的圖片格式：**
- JPG/JPEG
- PNG
- GIF
- BMP
- WEBP

### 📦 ZIP 文件

選擇包含圖片的 ZIP 壓縮包，系統會自動解壓到臨時目錄並掃描其中的圖片文件。

**ZIP 文件要求：**
- 標準 ZIP 格式
- 包含支持的圖片格式
- 文件擴展名必須為 `.zip`

## 使用方法

### 1. 選擇圖片來源

點擊 **"選擇圖片來源 (支持文件夾和 ZIP)"** 按鈕，在彈出的對話框中：

- 可以選擇單个或多个文件夾
- 可以選擇單个或多个 ZIP 文件
- 可以混合選擇文件夾和 ZIP 文件

**提示：** 按住 `Cmd` (macOS) 或 `Ctrl` (Windows) 可以多選。

### 2. 查看已選擇的來源

選擇後，所選的圖片來源會顯示在按鈕下方，每個來源會顯示：

- 類型圖標（📁 文件夾 / 📦 ZIP 文件）
- 文件/文件夾名稱
- 刪除按鈕（可以移除不需要的來源）

### 3. 開始處理

確保已選擇 Excel 文件和至少一個圖片來源後，點擊 **"開始處理"** 按鈕。

系統會：
1. 掃描所有圖片來源中的圖片文件
2. 根據 Excel 文件中的信息自動插入圖片
3. 顯示處理進度和統計信息

## 技術細節

### ZIP 文件處理

當選擇 ZIP 文件作為圖片來源時：

1. **自動解壓**：ZIP 文件會自動解壓到系統臨時目錄
2. **臨時目錄命名**：`image-auto-inserter-{時間戳}`
3. **自動清理**：
   - 組件卸載時自動清理所有臨時文件
   - 定時清理過期臨時文件（超過 1 小時）
   - 手動刪除來源時立即清理對應臨時文件

### ImageSourceManager

系統使用 `ImageSourceManager` 統一管理所有圖片來源：

```typescript
interface ImageSource {
  type: 'folder' | 'zip'           // 來源類型
  path: string                      // 原始路徑
  extractedPath?: string           // ZIP 解壓後的路徑
  name: string                      // 文件名
  addedAt: number                   // 添加時間
}
```

### 性能優化

- **異步處理**：所有文件操作均使用異步 API，不阻塞 UI
- **增量掃描**：只在添加新來源時掃描，不重複掃描
- **內存管理**：定時清理過期臨時文件，防止磁盤佔用過高

## 常見問題

### Q: ZIP 文件解壓失敗？

**可能原因：**
- ZIP 文件損壞
- ZIP 格式不支持（如 7z、RAR 等）
- 文件沒有讀取權限

**解決方案：**
- 重新壓縮 ZIP 文件
- 確保使用標準 ZIP 格式
- 檢查文件權限

### Q: 未找到圖片文件？

**可能原因：**
- 選擇的文件夾/ZIP 中沒有支持的圖片格式
- 圖片文件擴展名不正確
- 文件路徑包含特殊字符

**解決方案：**
- 確認文件夾/ZIP 中包含 JPG、PNG、GIF、BMP、WEBP 格式的圖片
- 檢查圖片文件擴展名是否正確
- 避免使用特殊字符命名文件

### Q: 臨時文件佔用過多磁盤空間？

**解決方案：**
- 處理完成後及時關閉應用（會自動清理）
- 手動刪除不需要的圖片來源
- 定期清理系統臨時目錄中的 `image-auto-inserter-*` 文件夾

## 最佳實踐

1. **使用文件夾而非 ZIP**：如果圖片已經在文件夾中，直接選擇文件夾更高效
2. **ZIP 適合傳輸**：ZIP 文件適合從他人處接收圖片包
3. **及時清理**：處理完成後，如果不再需要某個來源，點擊刪除按鈕
4. **預先檢查**：處理前確認圖片來源中包含所需的圖片文件
